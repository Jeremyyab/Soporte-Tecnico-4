Caso Práctico 1 — Seguridad en sistemas operativos

Aseguramiento de dispositivos móviles, Linux y macOS: bloqueos, biometría, cifrado y comandos de la CLI.

📌 Descripción del caso

Un colaborador de la empresa SeguraTI S.A. usa su teléfono, una laptop con Linux y una Mac para tareas de trabajo, todos sin medidas de seguridad. El equipo técnico debe proponer y aplicar la configuración mínima de seguridad para cada sistema operativo y documentar el proceso.

💡 Nota para el docente

Trabajen sobre dispositivos o máquinas virtuales de prueba. Documenten cada cambio con captura o anotación: la rúbrica valora la solución detallada del caso.

⚠️ Antes de empezar

Trabajen con cuentas de prueba. Nunca usen credenciales personales reales en un equipo compartido de laboratorio.

📱 Parte 1 — Asegurar el dispositivo móvil

  1. Activar el bloqueo de pantalla con PIN de 6 o más dígitos (no fechas ni 1234).
  2. Configurar la biometría (huella o rostro) con el PIN como respaldo.
  3. Activar la verificación en dos pasos (2FA) en la cuenta principal del dispositivo.
  4. Verificar que el cifrado del almacenamiento esté activo.
  5. Configurar el bloqueo automático por inactividad (30–60 s) y ocultar el contenido de las notificaciones.
  6. 📷 Captura 1: pantalla de seguridad con bloqueo y biometría configurados.

🐧 Parte 2 — Endurecer Linux

  1. Crear un usuario estándar: sudo adduser tecnico.
  2. Reservar la cuenta de administrador y usar sudo solo cuando se necesita.
  3. Revisar permisos de un archivo sensible con ls -l y ajustarlos: chmod 640 archivo.
  4. Corregir el propietario si corresponde: sudo chown usuario:grupo archivo.
  5. Activar el firewall: sudo ufw enable y verificar con sudo ufw status.
  6. Actualizar el sistema: sudo apt update && sudo apt upgrade.
  7. 📷 Captura 2: salida de ufw status y permisos del archivo.

🍏 Parte 3 — Asegurar macOS y verificar

  1. Activar FileVault: Preferencias del Sistema → Seguridad y privacidad → FileVault → Activar.
  2. Activar el firewall de aplicaciones.
  3. Activar las actualizaciones automáticas del sistema.
  4. Revisar los permisos de privacidad por aplicación.
  5. Verificar que los tres sistemas siguen funcionales tras los cambios.
  6. 📷 Captura 3: FileVault activo y firewall encendido.
⚠️ Escenario adicional (si terminan antes)

Configuren el borrado remoto y la localización del dispositivo móvil, y prueben la función desde la web de la cuenta.

🗣️ Preguntas de discusión para la retroalimentación

  1. ¿Por qué conviene trabajar con una cuenta estándar y elevar privilegios con sudo solo cuando se necesita?
  2. ¿Qué riesgo agrega la sincronización en la nube y cómo se mitiga?
  3. ¿En qué se diferencia el cifrado de disco (FileVault / LUKS) de una simple contraseña de inicio?
  4. ¿Qué harían si el usuario pierde el dispositivo móvil con datos del trabajo?
  5. ¿Por qué la biometría siempre se acompaña de un PIN de respaldo?
